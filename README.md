#Expense Tracker

## 1. 费用管理 
  这是整个应用的最顶层。在代码中，它对应的是程序的主入口和全局配置。

  数据存储逻辑：确定使用 JSON 格式作为本地数据库。

  环境初始化：程序启动时，检查是否存在 expenses.json 文件，如果不存在则初始化一个空列表。
### 1.1 费用管理模块
这一层级定义了程序如何处理数据流，即数据持久化层。

加载逻辑 (load_expenses)：负责将磁盘上的 JSON 字符串转换回 Python 的列表对象，供程序计算。

保存逻辑 (save_expenses)：负责将程序修改后的内存数据（列表/字典）重新序列化为文本，持久化存储到硬盘。

命令分发逻辑 (main)：利用 argparse 解析用户输入的指令（如 add, list），并将其路由到对应的功能函数。

#### 1.1.1 费用管理模块功能
  本模块实现了以下核心功能：

  添加支出 (add)：支持描述、金额及可选分类。系统自动分配唯一 ID 并记录当前日期。

  查看列表 (list)：以表格形式清晰展示所有历史支出记录。

  数据统计 (summary)：计算总支出，或按特定月份筛选统计当年支出。

  记录维护 (update/delete)：允许通过 ID 定位并修改或删除错误的记录。

  数据导出 (export)：将账单数据一键导出为 CSV 格式，方便在 Excel 中查看。
